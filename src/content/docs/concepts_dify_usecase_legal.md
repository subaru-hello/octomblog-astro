## 法律分野における AI 活用の位置付け

法律は「最終判断が人間（弁護士・当事者）にある」ことが大前提。Dify は以下の補助業務に特化して活用する。

```
Dify で自動化できる（補助的な）業務:
  ✅ 契約書の標準条項との差分チェック
  ✅ 契約書からのデータ抽出（当事者・金額・期間）
  ✅ 社内法令データベースへの Q&A
  ✅ 法律文書の平易な言葉への言い換え
  ✅ コンプライアンス確認チェックリストの生成

Dify でしてはいけないこと:
  ❌ 法的判断（「この契約は有効です」等）
  ❌ 訴訟戦略・弁護方針の決定
  ❌ 最終的な法的リスク評価
  → これらは必ず弁護士・法務担当者が判断する
```

## ユースケース1: 契約書リスク抽出ツール

### シナリオ

取引先から受け取った契約書（秘密保持契約・業務委託契約等）をアップロードすると、自社にとってリスクになりうる条項をフラグアップし、対応案を提示する。法務担当者の初期レビュー工数を削減する。

```
入力: 業務委託契約書（PDF）

出力:
  【契約書レビュー結果（ドラフト）】
  ※本レビューは参考情報です。最終判断は法務部門にて行ってください。

  🔴 要確認（重要）:
  第8条（損害賠償）
  「乙は甲に生じた一切の損害を賠償する」
  → 損害賠償の上限額の定めがありません。
    標準的な修正案: 「損害賠償額は委託料総額を上限とする」
    ← 法務の判断を仰ぐことを推奨

  🟡 要確認（中程度）:
  第12条（契約期間・更新）
  「本契約は自動更新される」
  → 更新拒絶の通知期限が60日前となっています。
    業界標準の30日前より長めです。
    変更交渉の余地があります。

  🟢 問題なし:
  第5条（秘密保持）: 標準的な条項
  第7条（著作権帰属）: 委託者帰属で適切

  【相手方への確認事項リスト】
  □ 損害賠償の上限額を追加する
  □ 自動更新の通知期限を30日に変更する交渉
```

### ワークフロー設計

```
Workflow 構成:

[Start]
  │ {{contract}}: 契約書 PDF
  │ {{contract_type}}: Select（NDA/業務委託/売買/賃貸等）
  ▼
[Doc Extractor]
  │ PDF テキスト抽出（条番号を保持）
  ▼
[Knowledge Retrieval: 自社標準契約書DB]
  │ 同種の標準契約書と比較
  │ → {{standard_clauses}}: 標準条項
  ▼
[LLM: リスク条項の特定]
  │ 標準からの逸脱・不利な条項を抽出
  │ → {{risk_clauses}}: JSON（条番号・リスク・修正案）
  ▼
[LLM: レビューレポート生成]
  │ 重要度別（🔴🟡🟢）に整理
  ▼
[End]
  出力: レビューレポート + 要確認リスト
```

## ユースケース2: 社内法令・規程 Q&A ボット

### シナリオ

就業規則・コンプライアンス規程・情報セキュリティポリシー等の社内規程をナレッジベースに登録。社員が「〇〇の場合はどうしたらいい？」と質問すると、該当箇所を引用して回答する。

```
社員: 「取引先から贈答品をいただいた場合、どうすればいいですか？
     5,000円相当のお菓子なのですが...」

ボット:
  「コンプライアンス規程第15条（利益供与の禁止）によると、
   取引先からの贈答品は原則として受領できません。
   
   ただし以下の場合は例外とされています:
   ・社会的慣習の範囲内（5,000円未満）かつ
   ・個人的な関係ではなく業務上の関係に基づくもの
   
   5,000円相当のお菓子は金額的には許容範囲ですが、
   念のため上長に報告することを推奨します。
   
   受領した場合は gifting-log@company.com まで報告してください。
   （出典: コンプライアンス規程 第15条・第16条）」
```

## ユースケース3: 契約書からの構造化データ抽出

### シナリオ

契約管理システムへの入力作業を自動化。契約書 PDF から当事者・契約金額・期間・更新条件を抽出して管理台帳に自動登録する。

```
入力: 業務委託契約書

出力（JSON → 契約管理システムへ自動登録）:
  {
    "contract_type": "業務委託契約",
    "party_a": {"name": "株式会社〇〇", "role": "委託者"},
    "party_b": {"name": "株式会社△△", "role": "受託者"},
    "contract_date": "2026-04-01",
    "effective_date": "2026-04-01",
    "expiry_date": "2027-03-31",
    "auto_renewal": true,
    "renewal_notice_days": 60,
    "contract_amount": 1200000,
    "payment_terms": "月末締め翌月末払い",
    "governing_law": "日本法",
    "jurisdiction": "東京地方裁判所"
  }
```

## 法律分野特有のプロンプト設計

```
必ず含めるべき制約文:

System Prompt の末尾に追記:

「重要な注意事項:
本ツールはリサーチ・整理の補助を目的としています。
法的判断・アドバイスを提供するものではありません。
契約の締結・変更・解除等の重要な判断は、
必ず弁護士または社内法務担当者に確認してください。
本ツールの出力を法的根拠として使用しないでください。」

出力の最後に自動付記するフッター（Template ノードで実装）:
  ---
  ※本レビューは AI による参考情報です。
  法的効力を持つものではありません。
  重要な判断は弁護士にご相談ください。
```
