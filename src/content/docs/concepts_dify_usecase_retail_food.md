## 小売・飲食業界での AI 活用の背景

小売・飲食業は利益率が薄く、人件費・オペレーションコストの削減が経営課題。一方でスタッフの入れ替わりも多く、「知識の属人化」が問題になりやすい。

Dify を活用することで：
- 新メニュー・商品説明文の作成コストをゼロに近づける
- アルバイトスタッフでも対応できる接客 Q&A を標準化する
- クレーム対応の品質にばらつきが出にくくなる

## ユースケース1: 飲食店メニュー説明文生成

### シナリオ

季節ごとに変わるメニューの説明文・POP コピー・SNS 投稿文を、食材・調理方法の情報から自動生成する。

```
入力（料理情報）:
  料理名: 黒毛和牛のビスク仕立て
  主な食材: 黒毛和牛もも肉・野菜ミックス・トマト
  調理法: 低温調理60度3時間→直火仕上げ
  特徴: 旨味を閉じ込めた柔らかな食感
  アレルゲン: 牛肉・なし（乳・卵不使用）

出力（同時生成）:

【メニュー表用（50文字）】
  丹精込めた黒毛和牛をフレンチの技法「ビスク」で包み込んだ
  シェフ渾身の一皿。とろける食感と凝縮した旨みをご堪能ください。

【卓上 POP 用（キャッチコピー）】
  「60度で3時間、愛情を込めて。」

【Instagram キャプション】
  🥩 今月のスペシャルメニューが登場！
  黒毛和牛を60度の低温でじっくり3時間…
  その後の直火仕上げが生む奇跡の食感✨
  乳・卵不使用なのでアレルギーの方もご安心を😊
  #黒毛和牛 #フレンチ #低温調理 #東京グルメ #恵比寿ランチ
```

### バッチ生成（全メニュー一括）

```
季節切り替え時に全メニューを一括更新:

[Start]
  │ {{menu_list}}: 全メニューのJSON配列
  ▼
[Iteration]
  │ 各料理に対して:
  │   [LLM: メニュー説明文生成]
  │   [LLM: SNS 投稿文生成]
  │ parallel_num: 5
  ▼
[Code: メニュー表 CSV 出力]
  ▼
[End]

→ 20品のメニュー説明文を3分で一括生成
```

## ユースケース2: クレーム対応メール自動生成

### シナリオ

顧客からのクレームメールを貼り付けると、適切な謝罪文・対応策・再発防止策を含む返信文案を自動生成する。担当者は確認・修正して送信するだけ。

```
入力（クレームメール）:
  「先日注文した商品が届いたのですが、箱が大きく破損していました。
  中身は無事でしたが、大切なプレゼント用に購入したのに
  このような状態で届いて非常に残念です。
  どのように対応していただけますか？」

出力（返信文案）:
  件名: Re: ご注文商品の梱包について（ご連絡）

  〇〇様

  この度はご注文いただきました商品の
  梱包状態についてご不満をおかけしてしまい、
  誠に申し訳ございません。

  大切なプレゼント用にお選びいただいたとのこと、
  そのような状況でお届けしてしまったことを
  深くお詫び申し上げます。

  つきましては、以下の対応をさせていただきたく存じます：
  ①新品を本日発送（追跡番号：後ほどご連絡いたします）
  ②破損品の返送は不要です
  ③ご不便をおかけした件として、500円クーポンをお送りいたします

  今後は梱包品質の確認を徹底してまいります。
  重ねてお詫び申し上げます。

  ───────────
  〇〇ショップ カスタマーサポート

  ---
  ※担当者確認事項:
  ・在庫状況を確認の上、発送日を修正してください
  ・クーポン付与の可否を上長に確認してください
```

### 対応難易度の自動分類

```
[Question Classifier]
  ├── 軽微なクレーム（商品の遅延・梱包）→ テンプレート + 自動返信案
  ├── 中程度（品質不良・欠品）           → 要確認事項リスト + 上長エスカレーション推奨
  └── 重大（健康被害・事故・法的リスク） → 即時上長通知・返信案を出さない

重大クレームの判定:
  「怪我」「病院」「弁護士」「返金要求」「SNS投稿する」
  等のキーワードが含まれる場合は自動分類
```

## ユースケース3: 店舗スタッフ向け商品Q&Aボット

### シナリオ

新人スタッフが「この商品の保存方法は？」「賞味期限が切れそうな商品はどう処理する？」等を Slack またはスマホから質問できるボット。

```
新人スタッフ（Slackで）:
  「@商品ボット 生ハムの開封後の保存方法を教えて」

ボット:
  「生ハムの開封後の保存方法 🥩
  
  1. ラップでしっかり包む（空気に触れないように）
  2. 冷蔵庫の低温部（0〜3℃）で保存
  3. 開封後は3〜5日以内に使い切る
  4. 使用後はラップを取り換える

  ⚠️ 異臭・変色が見られる場合は廃棄してください。
  
  （出典: 食品管理マニュアル p.24）」
```

### ナレッジベース登録コンテンツ

```
小売・飲食店での登録推奨コンテンツ:
  ✅ 商品・食材の管理マニュアル（保存・期限・廃棄基準）
  ✅ クレーム対応手順書（状況別の対応フロー）
  ✅ POSシステム・レジ操作マニュアル
  ✅ 衛生管理規程（HACCP対応）
  ✅ よくある質問集（FB・Instagramのコメント対応例）
  ✅ アレルゲン一覧（全商品分）
```

## コスト設計（小規模店舗向け）

```
小売・飲食は利益率が低いため、コストを最小化した設計が重要。

推奨構成（月数千円以内で運用）:
  モデル: GPT-4o mini（最安クラス）
  Dify: セルフホストまたは無料プラン
  
  月間利用量の試算:
    スタッフQ&A: 50回/日 × 30日 = 1,500回
    1回500トークン × 1,500回 = 750,000トークン
    GPT-4o mini: 約 $0.56/月（約85円）
  
  クレーム対応文案: 10通/月
    1通4,000トークン × 10 = 40,000トークン
    GPT-4o mini: 約 $0.03（約5円）
  
  合計: 月100円以下で全機能が使える
```
